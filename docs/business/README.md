# Xianyu AutoAgent 业务流程与用户意图文档

> 本文档是所有功能设计和代码修改的业务参考。每次改动前，先对照本文档确认是否符合业务目标。

---

## 1. 业务全貌

### 1.1 做什么

为闲鱼（Xianyu）卖家提供 24/7 AI 自动客服，帮卖家自动回复买家咨询、处理议价、解答技术问题，让卖家不用时刻盯着手机也能及时响应客户。

### 1.2 核心流程

```
买家在闲鱼发消息
    │
    ▼
系统通过 WebSocket 实时接收消息
    │
    ▼
消息解密 → 判断消息类型
    │
    ├── 订单消息（等待付款/交易关闭/等待发货）→ 仅记录日志
    ├── 系统消息（[xxx] 格式）→ 跳过，不回复
    ├── 正在输入状态 → 跳过
    ├── 卖家自己发的消息 → 检查是否为模式切换指令
    │
    └── 买家聊天消息 → 进入 AI 处理流程
                │
                ▼
        检查是否处于人工接管模式
            │
            ├── 是 → 仅存上下文，不回复
            │
            └── 否 → 获取商品信息 + 对话上下文
                    │
                    ▼
                意图识别（三级路由）
                    │
                    ├── 技术咨询 → TechAgent（带联网搜索）
                    ├── 议价还价 → PriceAgent（动态温度策略）
                    ├── 无需回复 → 不回复
                    └── 其他 → DefaultAgent
                    │
                    ▼
                安全过滤（屏蔽微信/QQ/支付宝等）
                    │
                    ▼
                发送回复 → 同步到 Supabase
```

### 1.3 核心约束

| 约束 | 说明 |
|---|---|
| 平台合规 | 所有对话必须在闲鱼平台内完成，AI 回复中不能出现微信、QQ、支付宝、银行卡、线下等引导用户离开平台的内容，否则可能导致封号 |
| Cookie 有效期 | 闲鱼的登录 Cookie 会过期或被风控，过期后机器人无法工作，需要人工更新 Cookie |
| 消息时效性 | 超过 5 分钟的历史消息会被丢弃，防止重复回复或回复过时内容 |
| 人工优先 | 卖家发送特定关键词（默认为"。"）可随时接管任何会话，AI 自动让路 |
| Token 有效期 | WebSocket 连接需要 Token 认证，Token 每 2 小时自动刷新，刷新时会短暂断连后自动重连 |

## 2. 数据分层

### 2.1 基础数据（很少变的）

```
账户（Account）
├── id（账户唯一标识，对应闲鱼 unb）
├── name（卖家名称，可在 Dashboard 编辑）
├── cookies_str（闲鱼登录凭证，触发风控时需要更新）
├── api_key（LLM API Key）
├── model_base_url（LLM 服务地址）
├── model_name（使用的模型名称）
└── status（online / offline / error）

商品（Item）
├── item_id（闲鱼商品 ID）
├── title（商品标题）
├── desc（商品描述）
├── price（商品价格）
├── skuList（规格列表：规格名、价格、库存）
└── quantity（总库存）
```

### 2.2 业务数据（经常操作的）

```
对话消息（Message）
├── chat_id（会话 ID，一个买家+一个卖家=一个会话）
├── user_id（发送者 ID）
├── item_id（关联商品 ID）
├── role（user=买家 / assistant=AI或卖家）
├── content（消息内容）
└── timestamp（时间戳）

议价计数（BargainCount）
├── chat_id（会话 ID）
├── count（该会话的累计议价次数）
└── last_updated（最后更新时间）

Prompt 提示词
├── account_id（所属账户）
├── type（classify / price / tech / default）
└── content（提示词内容）
```

## 3. AI 智能客服模块

### 3.1 核心概念

AI 智能客服是整个系统的核心，负责理解买家意图并生成合适的回复。系统采用多 Agent 架构，不同类型的问题由不同的专业 Agent 处理。

### 3.2 生命周期

```
买家发消息 → 意图识别 → 路由到对应 Agent → 生成回复 → 安全过滤 → 发送
```

### 3.3 用户操作意图

#### 买家咨询商品

**场景**：买家看到一个闲鱼商品，想了解更多信息，直接发消息问卖家
**操作**：
1. 买家发送消息（如"还在吗？""包邮吗？"）
2. AI 识别为一般咨询，路由到 DefaultAgent
3. DefaultAgent 结合商品信息和对话上下文生成回复

**设计要点**：
- 回复要自然，像真人卖家在聊天
- 必须基于商品实际信息回答，不能编造

#### 买家砍价

**场景**：买家想压低价格，反复还价
**操作**：
1. 买家发送还价消息（如"100 卖不卖？""便宜点"）
2. AI 识别为议价意图，路由到 PriceAgent
3. PriceAgent 根据议价次数动态调整策略（前几次坚挺，后面适当松口）
4. 系统记录议价次数，供后续轮次参考

**设计要点**：
- 动态温度策略：议价次数越多，AI 回复越灵活（temperature 从 0.3 递增到 0.9）
- 议价次数跟着会话走，换商品聊天不会重置

#### 买家问技术参数

**场景**：买家问商品规格、参数对比等需要专业知识的问题
**操作**：
1. 买家发送技术类问题（如"频响范围多少？""和 XXX 比怎么样？"）
2. AI 识别为技术意图，路由到 TechAgent
3. TechAgent 启用联网搜索能力，结合搜索结果和商品信息回复

**设计要点**：
- TechAgent 使用通义千问的联网搜索功能（`enable_search: true`）
- 回复要专业准确，不能凭空编造参数

#### 卖家接管会话

**场景**：某个买家的问题比较复杂，卖家想亲自回复
**操作**：
1. 卖家在闲鱼对话中发送切换关键词（默认为"。"）
2. 系统将该会话标记为人工接管模式
3. 后续该买家的消息只记录上下文，不触发 AI 回复
4. 卖家再次发送切换关键词，恢复 AI 自动回复

**设计要点**：
- 人工接管模式有超时机制（默认 1 小时），超时后自动恢复 AI 模式
- 切换关键词可通过环境变量 `TOGGLE_KEYWORDS` 自定义

## 4. WebSocket 通信模块

### 4.1 核心概念

系统通过 WebSocket 长连接与闲鱼服务器通信，实时收发消息。这是整个机器人的"耳朵和嘴巴"。

### 4.2 生命周期

```
获取 Token → 建立 WebSocket 连接 → 注册设备 → 心跳维持 → 收发消息 → Token 过期 → 刷新 Token → 重新连接
```

### 4.3 用户操作意图

#### 系统保持在线

**场景**：机器人需要 24/7 保持在线
**操作**：
1. 系统每 15 秒发送一次心跳包
2. 如果 20 秒内没收到心跳响应，判定连接断开
3. 断开后等 5 秒自动重连

**设计要点**：
- 心跳间隔和超时可通过环境变量调整
- Token 每 2 小时自动刷新，刷新时主动断开再重连

## 5. Dashboard 管理后台模块

### 5.1 核心概念

Dashboard 是一个 Web 管理界面，让卖家可以在浏览器里监控机器人状态、查看对话记录、修改配置，不需要动代码或命令行。

### 5.2 生命周期

```
卖家登录 → 查看账户状态 → 浏览对话记录 → 调整设置/提示词 → 保存
```

### 5.3 用户操作意图

#### 查看机器人状态

**场景**："我的机器人还在正常运行吗？今天回了多少消息？"
**操作**：
1. 打开 Dashboard 首页
2. 看到每个账户的在线状态（Online/Offline/Error）
3. 看到总消息数、今日消息数、错误数

#### 查看对话记录

**场景**："机器人跟买家聊了什么？有没有回答不好的地方？"
**操作**：
1. 进入对话记录页
2. 按时间浏览所有对话
3. 可以筛选特定账户的对话

#### 修改配置和提示词

**场景**："我想让机器人的回复风格更热情一些"
**操作**：
1. 进入设置页
2. 编辑 Prompt 提示词（分类/议价/技术/默认四种）
3. 修改 Cookie、API Key 等配置
4. 保存后 Python 端下次加载时生效

## 6. 模块间关系

```
Dashboard (1)
  ├── Account (1..N)        每个 Dashboard 管理多个闲鱼账户
  │     ├── Prompts (4)     每个账户 4 套提示词
  │     ├── Conversations (0..N)  对话记录
  │     └── Logs (0..N)     运行日志
  │
  └── Python Bot (1)        每个账户对应一个 Bot 实例
        ├── WebSocket (1)   一个长连接
        ├── ContextManager (1)  本地 SQLite 对话管理
        │     ├── Messages (0..N)
        │     ├── BargainCounts (0..N)
        │     └── Items (0..N)
        └── Agents (4)      四个专业 Agent
              ├── ClassifyAgent  意图分类
              ├── PriceAgent     议价处理
              ├── TechAgent      技术咨询
              └── DefaultAgent   默认回复
```
