# AI 智能客服 Agent 模块

> 一句话总结：这个模块负责理解买家说了什么、决定怎么回复、生成自然的回复内容。

---

## 1. 核心概念

AI 智能客服采用多 Agent 架构。每条买家消息先经过意图识别路由器（IntentRouter），决定由哪个专业 Agent 处理，然后由对应 Agent 生成回复。

四个 Agent 各有分工：
- **ClassifyAgent**：意图分类，内部使用，不直接生成回复
- **PriceAgent**：处理议价还价，根据议价次数动态调整策略
- **TechAgent**：处理技术参数咨询，启用联网搜索
- **DefaultAgent**：处理其他所有一般咨询

---

## 2. 意图路由（三级策略）

```
买家消息输入
    │
    ▼
第 1 级：技术类关键词匹配
    │  命中 ['参数','规格','型号','连接','对比'] → tech
    │  命中正则 r'和.+比' → tech
    │
    ├── 命中 → 直接返回 tech
    │
    ▼ 未命中
第 2 级：价格类关键词匹配
    │  命中 ['便宜','价','砍价','少点'] → price
    │  命中正则 r'\d+元', r'能少\d+' → price
    │
    ├── 命中 → 直接返回 price
    │
    ▼ 未命中
第 3 级：大模型分类（ClassifyAgent）
    │  把消息 + 商品信息 + 上下文发给 LLM
    │  LLM 返回 price / tech / default / no_reply
    │
    └── 返回 LLM 判定的意图
```

**设计要点**：
- 技术类优先于价格类判断，因为"参数"类问题误判代价更高
- 关键词/正则是快速通道，大模型是兜底
- `no_reply` 意图表示不需要回复（如买家说"好的""谢谢"）

## 3. Agent 生成流程

### 3.1 消息构建

所有 Agent 共用相同的消息链构建逻辑：

```
System Message:
  "【商品信息】{item_desc}
   【你与客户对话历史】{context}
   {system_prompt}"

User Message:
  "{买家发的消息}"
```

### 3.2 各 Agent 特殊逻辑

| Agent | 温度 | 特殊能力 | 附加信息 |
|---|---|---|---|
| ClassifyAgent | 0.4 | 无 | 无 |
| PriceAgent | 0.3 ~ 0.9（动态） | 议价轮次感知 | 追加 `▲当前议价轮次：{count}` |
| TechAgent | 0.4 | 联网搜索（`enable_search: true`） | 无 |
| DefaultAgent | 0.7 | 无 | 无 |

### 3.3 安全过滤

每条 AI 回复生成后都经过安全过滤：

| 检测内容 | 处理 |
|---|---|
| 包含"微信"、"QQ"、"支付宝"、"银行卡"、"线下" | 替换为 `[安全提醒]请通过平台沟通` |
| 不包含上述内容 | 原样输出 |

## 4. Prompt 加载优先级

```
Supabase 云端 Prompts（最高优先）
    │ 没有？
    ▼
本地 prompts/ 目录下的文件（如 price_prompt.txt）
    │ 没有？
    ▼
本地 prompts/ 目录下的示例文件（如 price_prompt_example.txt）
```

**四种 Prompt 类型**：
1. `classify` → `classify_prompt.txt` — 意图分类的指令
2. `price` → `price_prompt.txt` — 议价场景的人设和策略
3. `tech` → `tech_prompt.txt` — 技术咨询的人设和策略
4. `default` → `default_prompt.txt` — 一般咨询的人设和策略

---

## 5. 议价温度策略

```
温度 = min(0.3 + 议价次数 × 0.15, 0.9)

议价次数 0 → 温度 0.30（很坚定）
议价次数 1 → 温度 0.45
议价次数 2 → 温度 0.60
议价次数 3 → 温度 0.75
议价次数 4+ → 温度 0.90（比较灵活）
```

**设计意图**：
- 买家刚开始砍价时，AI 态度坚定，维护价格
- 多次议价后，AI 适当松口，展现灵活性
- 温度上限 0.9，防止回复过于随机不可控
