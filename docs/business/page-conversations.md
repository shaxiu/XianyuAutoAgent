# 对话记录页 — 用户目标与信息规格

> 一句话总结：这个页面是让卖家查看机器人和买家的所有对话内容。

---

## 1. 用户场景

**场景 A：检查回复质量**
- "我想看看机器人都是怎么回复买家的，有没有回答得不好的地方"
- 用户目标：逐条浏览对话，评估 AI 回复是否得体

**场景 B：查找特定对话**
- "有个买家说机器人回复了奇怪的内容，我要找到那条对话看看"
- 用户目标：按账户筛选，找到特定对话记录

**场景 C：了解客户关注点**
- "最近买家都在问什么问题？有没有什么共性需求？"
- 用户目标：浏览近期对话，发现买家关注的模式

---

## 2. 页面信息架构（从上到下）

### 区域 1：标题 + 筛选（页面顶部）

包含：
- 页面标题 "Conversations"
- 账户选择器（下拉框，筛选特定账户的对话）

### 区域 2：对话列表（主区域）

**表格形式展示，按时间倒序。**

每条记录包含：

| 字段 | 说明 |
|---|---|
| 时间 | 消息发送时间 |
| 会话 ID | chat_id，同一买家的连续对话 |
| 商品 | 关联的商品 ID 和标题 |
| 角色 | user（买家）/ assistant（AI 回复） |
| 内容 | 消息文本 |
| 意图 | AI 识别的意图（price/tech/default/no_reply） |

---

## 3. 页面角色定位

| 能做的 | 不能做的 |
|---|---|
| 查看所有对话记录 | 不能删除对话 |
| 按账户筛选对话 | 不能在这里直接回复买家 |
| 查看 AI 识别的意图 | 不能修改 AI 的回复内容 |
