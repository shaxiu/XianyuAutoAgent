# Dashboard 首页 — 用户目标与信息规格

> 一句话总结：这个页面是让卖家快速了解所有闲鱼账户的机器人运行状态。

---

## 1. 用户场景

**场景 A：日常巡检**
- "我早上起来，想看看昨晚机器人有没有正常运行"
- 用户目标：快速确认所有账户在线，没有报错

**场景 B：发现问题**
- "收到买家投诉说没收到回复，是不是机器人挂了？"
- 用户目标：快速定位哪个账户出了问题

---

## 2. 页面信息架构（从上到下）

### 区域 1：标题栏（页面顶部）

**始终可见。**

包含：
- 页面标题 "Dashboard"
- 侧边导航栏（Dashboard / Conversations / Logs / Settings）
- 主题切换按钮（明/暗模式）

### 区域 2：账户卡片网格（主区域）

**每个闲鱼账户一张卡片，2 列网格布局。**

每张卡片包含：

| 信息 | 位置 | 说明 |
|---|---|---|
| 账户名称 | 卡片标题 | 可在 Settings 页编辑 |
| 在线状态 | 标题右侧 Badge | Online(绿) / Offline(灰) / Error(红) |
| Total Messages | 卡片内左列 | 该账户的总消息数 |
| Today | 卡片内中列 | 今日消息数 |
| Errors | 卡片内右列（红色） | 错误日志数 |

**交互规则**：

| 当前条件 | 显示/行为 |
|---|---|
| 数据加载中 | 显示 "-" 占位 |
| 账户在线 | Badge 显示绿色 "Online" |
| 账户离线 | Badge 显示灰色 "Offline" |
| 账户出错 | Badge 显示红色 "Error" |
| 没有账户 | 页面空白（无卡片） |

---

## 3. 页面角色定位

| 能做的 | 不能做的 |
|---|---|
| 查看所有账户状态 | 不能在这个页面编辑配置 |
| 查看消息统计概览 | 不能查看具体对话内容 |
| 快速判断是否有异常 | 不能直接重启 Bot |
