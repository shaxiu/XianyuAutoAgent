# 设置页 — 用户目标与信息规格

> 一句话总结：这个页面是让卖家配置机器人的行为参数和 AI 提示词。

---

## 1. 用户场景

**场景 A：调整 AI 回复风格**
- "我觉得机器人回复太生硬了，想让它更热情一些"
- 用户目标：编辑提示词，调整 AI 的人设和回复策略

**场景 B：更新 Cookie**
- "机器人掉线了，需要更新 Cookie 让它重新连上"
- 用户目标：粘贴新的 Cookie 字符串并保存

**场景 C：切换 AI 模型**
- "想试试用另一个大模型，看看效果怎么样"
- 用户目标：修改 API Key、模型地址、模型名称

**场景 D：管理多账户**
- "我要给另一个闲鱼号也配上自动客服"
- 用户目标：切换账户，为不同账户配置不同参数

---

## 2. 页面信息架构（从上到下）

### 区域 1：标题 + 账户选择（页面顶部）

包含：
- 页面标题 "Settings"
- 账户选择器（选择要配置哪个账户）

### 区域 2：基本设置（Tab 1）

| 配置项 | 类型 | 说明 |
|---|---|---|
| 账户名称 | 文本输入框 | 显示在 Dashboard 卡片上的名称 |
| Cookie | 长文本输入框 | 闲鱼登录凭证，掉线时需要更新 |
| API Key | 文本输入框 | LLM API 密钥 |
| Model Base URL | 文本输入框 | LLM 服务端点地址 |
| Model Name | 文本输入框 | 使用的模型名称 |

### 区域 3：提示词设置（Tab 2）

四个独立的文本编辑区：

| 提示词类型 | 用途 |
|---|---|
| Classify Prompt | 意图分类指令：告诉 AI 怎么判断买家想干什么 |
| Price Prompt | 议价策略：告诉 AI 怎么应对砍价 |
| Tech Prompt | 技术咨询：告诉 AI 怎么回答技术问题 |
| Default Prompt | 默认回复：告诉 AI 怎么处理一般咨询 |

**交互规则**：

| 操作 | 行为 |
|---|---|
| 修改后点保存 | 写入 Supabase，Bot 下次初始化时生效 |
| 不保存直接离开 | 修改丢失 |
| 清空某个 Prompt | Bot 回退使用本地文件中的 Prompt |

---

## 3. 页面角色定位

| 能做的 | 不能做的 |
|---|---|
| 修改所有配置参数 | 不能直接重启 Bot |
| 编辑 AI 提示词 | 不能实时预览 AI 回复效果 |
| 更新 Cookie | 不能自动检测 Cookie 是否有效 |
| 切换不同账户配置 | 不能添加新账户（需在 Supabase 中手动创建） |
